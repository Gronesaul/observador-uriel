Guía completa de deploy — Backend en Railway · Frontend en Netlify

1

Subir el proyecto a GitHub

Primero debes tener el repositorio en GitHub. Abre una terminal dentro de la carpeta ObservadorUriel y ejecuta:

git init
git add .
git commit -m "ObservadorUriel - deploy inicial"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/observador-uriel.git
git push -u origin main
⚠️ Reemplaza TU_USUARIO con tu usuario de GitHub. Si aún no tienes el repo creado, créalo primero en github.com (sin inicializar con README).
2

Backend en Railway (FastAPI)

2.1 — Crear el proyecto
  1. Ve a railway.app → New Project
  2. Elige Deploy from GitHub repo → selecciona observador-uriel
  3. Railway detecta el railway.toml automáticamente
Si Railway no detecta el backend, configura manualmente:
Root Directory: backend
Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT
2.2 — Agregar PostgreSQL
  1. Dentro de tu proyecto en Railway → + New → Database → Add PostgreSQL
  2. Railway vincula DATABASE_URL automáticamente ✅
2.3 — Variables de entorno

Ve a la sección Variables de tu servicio backend y agrega:

VariableValor
SECRET_KEYGenera una clave larga (ver abajo)
ALGORITHMHS256

Para generar SECRET_KEY, ejecuta en terminal:

python3 -c "import secrets; print(secrets.token_hex(32))"
2.4 — Obtener la URL del backend

Una vez desplegado, Railway te da una URL así:

https://observador-uriel-production.up.railway.app
✅ Copia esta URL. La necesitas para configurar el frontend en el Paso 3.
3

Frontend en Netlify (React/Vite)

3.1 — Conectar con GitHub
  1. Ve a netlify.com → Add new site → Import from Git
  2. Selecciona tu repo observador-uriel
  3. Netlify detecta el netlify.toml. Verifica que quede así:
CampoValor
Base directoryfrontend
Build commandnpm run build
Publish directoryfrontend/dist
3.2 — Variable de entorno en Netlify

Ve a Site settings → Environment variables → Add variable:

KeyValue
VITE_API_URLhttps://observador-uriel-production.up.railway.app
⚠️ Usa la URL real de Railway que copiaste en el Paso 2.4 — sin barra al final.
3.3 — Deploy

Haz clic en Deploy site. Netlify construye e instala todo automáticamente en ~2 minutos.

✅ Tu site quedará en algo como: https://observador-uriel.netlify.app
4

Configurar CORS (conectar Railway ↔ Netlify)

Una vez tengas la URL de Netlify, vuelve a Railway y agrega esta variable en el backend:

VariableValor
FRONTEND_URLhttps://observador-uriel.netlify.app
⚠️ Después de agregar la variable, Railway redespliega automáticamente el backend.

Solución de problemas comunes

❌ Error 500 / "Internal Server Error"
Verifica que DATABASE_URL está configurada en Railway. Revisa los logs del servicio en Railway → sección Deployments.
❌ Error CORS (blocked by CORS policy)
Verifica que FRONTEND_URL en Railway tiene exactamente la misma URL que Netlify, sin barra al final. Redespliega el backend.
❌ Página en blanco en Netlify
Verifica que VITE_API_URL está configurada en las variables de entorno de Netlify. Reconstruye el site desde Netlify → Deploys → Trigger deploy.
❌ WhatsApp no abre el mensaje
El teléfono del acudiente debe estar registrado en la ficha del estudiante. Debe ser número colombiano de 10 dígitos (sin +57). Edítalo desde la ficha del estudiante.
❌ No aparecen los 124 estudiantes
La función importar_estudiantes_inicial() en main.py solo carga si la tabla está vacía. Si tienes datos corruptos, borra la base de datos en Railway y redespliegua.
